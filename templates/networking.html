🌐 Networking Basics

Let’s explore the world of networking, the foundation of all digital communication in cybersecurity.

✅ What is Networking?


  • 🧑‍🤝‍🧑 In career development, networking means creating professional relationships — where people connect to share knowledge, advice, and opportunities for growth.
  • 💻 In technology, networking refers to linking multiple devices like computers and servers to share data and communicate efficiently.
  • 🔐 In the context of cybersecurity, understanding how networks behave is crucial — it helps detect unusual activities, prevent attacks, and respond to security threats quickly.

🔑 Key Concepts of Networking

  • Nodes: Devices connected to a network, such as computers, servers, printers, or routers.
  • Protocols: Rules that govern how data is transmitted and received (e.g., TCP/IP, HTTP, FTP).
  • Network Topology: The physical/logical arrangement of nodes (bus, star, ring, mesh, tree).
  • Routers: Devices that connect different networks and direct data traffic.
  • Switches: Devices that connect multiple devices in the same network to communicate internally.
  • Wireless Access Points (WAPs): Devices that allow wireless connections to a network.
  • IP Addresses: Unique identifiers used to locate and communicate with devices on a network.
  • Firewall: A security system that filters traffic to prevent unauthorized access.
  • LAN (Local Area Network): Covers a small area like a home, school, or office.
  • WAN (Wide Area Network): Covers a larger area like cities or countries (e.g., the Internet).

✅ OSI Model (7 Layers)

Click below to expand and learn about each OSI layer

  1. Physical Layer:

    2. The Physical Layer is the first and lowest level of the OSI model. It handles the actual movement of raw binary data (bits) through physical connections like cables or wireless signals. This layer defines the hardware setup required for sending and receiving signals across devices.

    Key Functions:

    • Bit Transmission: Transfers raw 0s and 1s through the network medium.
    • Signal Encoding: Converts bits into electrical, light, or radio signals for transmission.
    • Physical Medium: Defines specs of cables (Ethernet, fiber) and connectors (RJ-45, etc.).
    • Hardware Devices: Includes NICs, hubs, repeaters that handle actual data movement.
    • Synchronization: Keeps sender and receiver in sync during communication.
    • Media Access (optional): Controls how devices share the same medium if needed.

    Examples:

    • Cables: Like Ethernet and fiber optic cables used in LANs and WANs.
    • Wireless: Involves radio waves and frequencies used in Wi-Fi or mobile networks.
    • Connectors: RJ-45 (Ethernet), fiber optic plugs, etc.
    • Hubs & Repeaters: Devices that extend or amplify signals at the physical level.

    💡 Summary: The Physical Layer forms the foundation for all network communication. Without it, no data can physically move between devices.

  2. Data Link Layer (Layer 2):

    3. The Data Link Layer is the second layer in the OSI model. It helps devices on the same local network communicate smoothly by ensuring that data gets from one device to another without errors.

    ✨ Key Functions:

    • ✔ Node-to-Node Delivery: Ensures data is passed directly between two connected devices (like PC to switch).
    • 🧩 Framing: Breaks down data into frames, which include headers (like MAC addresses) and error-checking info.
    • 🛡 Error Detection & Correction: Detects and sometimes fixes errors that happen during data transfer.
    • 🔁 Flow Control: Controls how fast data is sent, so the sender doesn’t overwhelm the receiver.
    • ⚖ Access Control: Manages how multiple devices share the same cable or medium without collisions.

    📚 Sublayers:

    • LLC (Logical Link Control): Handles error checking and identifies protocols from upper layers.
    • MAC (Media Access Control): Decides which device can use the medium and when (like Ethernet rules).

    🌐 Common Protocols:

    • Ethernet: Used in LANs for wired communication.
    • HDLC: A protocol that ensures smooth communication over point-to-point links.
    • PPP: Connects two nodes directly, often used in dial-up connections.
    • Frame Relay: A protocol used in wide area networks (WANs) for fast communication.

    🚀 Summary: The Data Link Layer creates a safe path between devices on the same network, turning raw data into manageable frames and making sure everything arrives accurately.

  3. Network Layer (Layer 3):

    4. The Network Layer (Layer 3) of the OSI model plays a vital role in transferring data between different networks. It manages the logical addressing system using IP addresses and ensures that data reaches the correct destination—even across the internet. This layer decides the best route for data packets and handles traffic control. Routers operate at this layer to connect different networks and direct traffic efficiently.

    🌐 Key Responsibilities:

    • Routing: Determines the optimal path for data to travel through multiple routers and networks to reach the destination.
    • Logical Addressing: Assigns IP addresses (IPv4 and IPv6) to identify the source and destination of packets.
    • Packet Forwarding: Sends packets from one network to another based on routing decisions.
    • Fragmentation: Breaks large packets into smaller ones if needed, especially if the receiving network has size limitations.
    • Reassembly: Collects fragmented packets and assembles them at the destination.

    🔁 Layer Interaction:

    • Receives data from: The Transport Layer (e.g., TCP, UDP)
    • Sends data to: The Data Link Layer (for physical transmission)

    🛠️ Real-Life Example:

    Imagine sending a parcel from Chennai to New York. The parcel (your data) is labeled with a destination address (IP). The courier service (routers) checks the address and finds the best route. If it’s too big for one vehicle (MTU), it’s broken into smaller packets (fragmentation) and reassembled when it arrives.

    📦 Network Layer Devices:

    • Routers: Core devices that read IP headers and forward data packets.
    • Layer 3 Switches: Can perform routing in addition to switching.

    📚 Common Protocols in Network Layer:

    • IP (Internet Protocol): Main protocol for addressing and routing data (IPv4 & IPv6).
    • ICMP (Internet Control Message Protocol): Used for error messages, diagnostics (e.g., ping).
    • IGMP (Internet Group Management Protocol): Manages multicast group communication.
    • IPsec (Internet Protocol Security): Adds encryption and authentication to IP communication for secure data transfer.

    🔐 Security in Network Layer:

    • Filters IP packets based on addresses and ports (via firewalls).
    • Uses VPN protocols (like IPsec) for private and encrypted communication over public networks.

    🧠 Fun Fact:

    The Internet is basically a giant Network Layer system! Billions of devices find each other using IP and routers—just like a digital GPS system.

    🎯 Summary:

    The Network Layer acts like a GPS and courier system for the Internet. It finds the shortest and safest path for your data, making sure it reaches the correct device—even across the world!

  4. Transport Layer:

    5. The Transport Layer (Layer 4 of the OSI Model) plays a key role in delivering data across networks from one system to another — securely, correctly, and in sequence. It acts like a reliable delivery service between apps running on different devices. This layer is responsible for setting up connections, managing how much data is sent, and checking that everything arrives intact.

    🔹 Core Functions:

    • Segmentation and Reassembly: Breaks large data into small segments for sending and reassembles them at the destination.
    • Connection Management: Establishes, maintains, and terminates reliable communication sessions (TCP).
    • Flow Control: Prevents network overload by controlling how much data is sent at a time.
    • Error Detection and Correction: Identifies lost or corrupted data and triggers automatic re-transmission.
    • Reliable Delivery: Ensures data reaches in proper order without duplication using sequence numbers and acknowledgments.
    • Multiplexing: Uses port numbers to allow multiple apps to run network tasks simultaneously on the same device.

    🔹 Key Protocols in Transport Layer:

    • TCP (Transmission Control Protocol):
      • Connection-oriented and reliable
      • Ensures all data reaches in correct order
      • Used in: Web browsing (HTTP/HTTPS), Email (SMTP), File Transfer (FTP)
    • UDP (User Datagram Protocol):
      • Connectionless and faster than TCP
      • No guarantee of delivery, used where speed is more important
      • Used in: Video streaming, Voice calls (VoIP), Online games

    🔹 Real-Time Examples:

    • 📥 Downloading a file – TCP ensures each part arrives and is rebuilt correctly.
    • 🎮 Playing an online game – UDP sends quick updates even if a few are lost.

    ✅ Summary: The Transport Layer guarantees smooth communication between apps across networks. It provides the backbone for reliable internet functions, handling delivery, error control, and data management between sender and receiver.

  5. Session Layer:

    6. The Session Layer is the fifth layer of the OSI (Open Systems Interconnection) model. It is responsible for establishing, managing, and terminating communication sessions between two devices or applications. While layers below it focus on how data is moved, the session layer controls how long and how reliably two devices talk to each other during a session.

    🔹 Key Functions of the Session Layer:

    • Session Establishment: It sets up a communication channel between sender and receiver before data is transferred. It negotiates parameters like which ports to use and session protocols.
    • Session Maintenance: Once established, it maintains the connection — ensuring both sides stay connected without disruption. If the connection drops, it can attempt to recover and resume the session.
    • Synchronization: The session layer places "checkpoints" or markers in the data stream so that if a session is interrupted, data transmission can resume from the last checkpoint — not from the beginning. This is especially useful in long file transfers.
    • Session Recovery: If there is a failure or disconnection, the session layer can restart the session from the last saved point instead of starting from scratch.
    • Session Termination: It gracefully ends the session after communication is complete, ensuring both parties agree to end the connection properly and that all data was received.

    🔹 Real-Life Example:

    Think of a WhatsApp voice call. Before you speak, a connection is established (Session Establishment). While talking, the app ensures your voice is sent and received smoothly (Maintenance). If the call drops, it shows “Reconnecting...” and continues from where it left off (Recovery). When you hang up, the session is closed properly (Termination). That’s the job of the session layer!

    🔹 Common Protocols Used:

    • NetBIOS (Network Basic Input/Output System): Used in local networks for session-level services.
    • RPC (Remote Procedure Call): Lets one program call functions or services on another computer as if it were local.
    • SQL: In database sessions, SQL keeps connections between client and server to allow data retrieval and updates.
    • ASP Session State / PHP Session: In web development, sessions are used to remember user actions, logins, and more during interaction with a website.

    🔹 Importance in Cybersecurity:

    • Session Hijacking Protection: Proper session management prevents attackers from stealing active sessions.
    • Session Timeout: Ends idle sessions automatically to reduce risk of unauthorized access.
    • Secure Session IDs: Ensures random and unique IDs are used to avoid predictable sessions in web apps.

    🔹 More Use Cases:

    • Video conferencing software maintaining a stream session (Zoom, Google Meet).
    • Multiplayer online games keeping player data in sync.
    • FTP file transfer sessions with checkpoints in case of disconnection.
    • Banking websites that log you out after inactivity.

    ✅ Summary: The Session Layer plays a vital role in making communication between devices smooth and reliable. Without it, applications would struggle to manage long conversations, resume broken connections, or ensure secure and synchronized communication. It is especially useful in applications where continuous communication is essential — such as video calls, data transfer, and real-time collaboration.

  6. Presentation Layer:

    7. The Presentation Layer is the sixth layer of the OSI model. It works as the system’s translator, ensuring that data from one device can be properly understood by another. This layer is responsible for data formatting, encryption, decryption, and compression — making data usable, secure, and efficient across platforms.

    🔹 Key Responsibilities:

    • 📝 Data Formatting and Translation: Converts data into a standard format so different operating systems and applications can interpret and use the information correctly. For example, it ensures a Windows system can read data from a Linux system.
    • 🔐 Encryption and Decryption: This layer ensures data security by encrypting data before transmission and decrypting it upon receipt. It’s responsible for converting plain text into cipher text and vice versa.
    • 📦 Compression and Decompression: Reduces the size of data to make it faster to transmit over the network. On the receiving side, the data is decompressed back to its original form.

    🔹 Common Standards and Protocols:

    • Character Encoding: ASCII, EBCDIC – used to represent text characters in binary.
    • Image & Video Formats: JPEG, PNG, MPEG, TIFF – define how multimedia content is formatted and displayed.
    • Security Protocols: SSL (Secure Socket Layer), TLS (Transport Layer Security) – used to provide secure data encryption and integrity.

    🔹 Real-World Examples:

    • When you watch a video online, the presentation layer helps format the video for your device and compress it to stream efficiently.
    • During a secure website transaction (https://), the presentation layer helps encrypt your personal data (like passwords).
    • When transferring files, the system ensures both sides can understand the format using encoding like UTF-8 or ASCII.

    🔹 Importance in Cybersecurity:

    • Ensures that data remains confidential through encryption.
    • Helps protect data from being intercepted in a readable form.
    • Supports secure communication between web browsers and servers using TLS/SSL.

    ✅ Summary: The Presentation Layer guarantees that data shared across networks is correctly formatted, compressed for speed, and protected through encryption. Without this layer, systems wouldn’t understand each other’s data, and secure communication would be impossible.

  7. Application Layer:

    8. The Application Layer is the seventh and final layer of the OSI model. It directly interacts with end-user software, acting as a bridge between the user and the network. This layer enables applications to communicate over the internet and other networks by providing essential services such as email, file transfer, and web browsing.

    🔹 Key Functions and Responsibilities:

    • 👤 User Interface: This layer provides the tools and graphical interface users interact with—like browsers, email clients, and file transfer tools.
    • 🔄 Protocol Support: Implements standard protocols such as HTTP, FTP, SMTP, and DNS to facilitate communication between applications.
    • 🔐 Data Services: Manages encryption, error control, and data formatting before it is passed down the network stack or received by the user.
    • 📡 Network Services: Enables vital services such as:
      • File transfers
      • Email communication
      • Remote system access
      • Domain name resolution

    🔹 Common Protocols Used:

    • HTTP (Hypertext Transfer Protocol): Used to browse websites.
    • FTP (File Transfer Protocol): Transfers files between systems.
    • SMTP (Simple Mail Transfer Protocol): Sends emails.
    • DNS (Domain Name System): Resolves domain names to IP addresses.
    • Telnet & SSH: Provide remote access to systems.

    🔹 Real-World Examples:

    • When you send an email via Gmail or Outlook, the Application Layer handles the process using SMTP.
    • When you open a website, your browser communicates with a server using HTTP or HTTPS.
    • When downloading files via FTP clients like FileZilla, this layer manages the protocol communication.

    🔹 Importance in Cybersecurity:

    • Application-level vulnerabilities are a major target for hackers (e.g., XSS, SQL injection).
    • Secure protocols like HTTPS help protect sensitive data from interception.
    • Authentication, data validation, and session management are handled here.

    ✅ Summary: The Application Layer ensures users and software can effectively communicate over a network. It is the most visible layer to the user and a critical point for implementing security and data handling features.

    🎯 OSI in Simple Words: Think of the OSI model as a 7-step delivery system—from the physical wire to the app you’re using. The Application Layer is where it all comes together for the user to see, click, and connect.

🛡 Everything You Need to Know About Networking for Cybersecurity

✅ 1. Basics of Computer Networking

➤ What is a Network?

A network is a collection of computers and devices connected to share data and resources.

➤ Types of Networks:
  • LAN (Local Area Network): Small area like home or school.
  • WAN (Wide Area Network): Large areas (Internet).
  • MAN (Metropolitan): Covers city-wide area.
  • PAN (Personal): Small personal range (Bluetooth, Hotspot).
➤ Network Topologies:
  • ⭐ Star: All devices connect to a central hub/switch.
  • 🚌 Bus: Devices connected in a line to a single cable.
  • 🔁 Ring: Devices form a circle.
  • 🔗 Mesh: Every device connects to every other device.
➤ OSI Model (7 Layers):
  1. Physical – Hardware (cables, switches)
  2. Data Link – MAC addresses
  3. Network – IP addresses
  4. Transport – TCP/UDP
  5. Session – Connection management
  6. Presentation – Data formatting, encryption
  7. Application – Protocols like HTTP, FTP
➤ TCP/IP Model:
  1. Network Interface
  2. Internet
  3. Transport
  4. Application
➤ IP Addressing:
  • IPv4: 192.168.0.1
  • IPv6: 2001:0db8:85a3::8a2e:0370:7334
➤ Subnetting Example:
  • IP: 192.168.1.10/24
  • Subnet Mask: 255.255.255.0
  • Host range: 192.168.1.1 – 192.168.1.254
➤ MAC Address:

A physical address like 00:1A:2B:3C:4D:5E

➤ Ports and Protocols:
  • HTTP – 80
  • HTTPS – 443
  • DNS – 53
  • SSH – 22
➤ Linux Commands:
ifconfig     # Shows IP and MAC address
ip a         # Modern command to show interfaces

✅ 2. Network Devices

  • Router: Connects different networks.
  • Switch: Connects devices in the same network.
  • Hub: Broadcasts to all devices.
  • Bridge: Connects two LANs.
  • Gateway: Connects different protocols.
  • Firewall: Filters traffic.
  • Modem: Internet connection.
  • Access Point: Provides Wi-Fi signal.

✅ 3. Network Protocols and Services

➤ TCP vs UDP:
  • TCP: Reliable, slower, ordered (e.g., HTTP, FTP)
  • UDP: Fast, no guarantee, good for video/audio
➤ Important Protocols:
  • ARP: Converts IP to MAC
  • ICMP: Used in ping & traceroute
  • DNS: Resolves domain to IP
  • DHCP: Assigns IP addresses automatically
  • NAT: Translates private to public IP
➤ Linux Commands:
ping google.com
traceroute google.com
arp -a

✅ 4. Network Security Fundamentals

➤ CIA Triad:
  • Confidentiality: No unauthorized access
  • Integrity: Data is accurate and untampered
  • Availability: Services accessible
➤ Common Attacks:
  • DoS/DDoS: Overload a system
  • MITM: Hacker intercepts communication
  • ARP Spoofing: Fakes MAC address
  • DNS Poisoning: Wrong IP for a domain
➤ Linux Security Tools:
netstat -tuln
ss -tuln
iptables -L
ufw status

✅ 5. Tools for Network Security

  • Wireshark: Packet capture
  • Nmap: Port scanning
  • Netstat: Network connection viewer

➤ WiFi Encryption Standards:
  • WEP: Weak
  • WPA: Better
  • WPA2: Strong

  • Evil Twin: Fake WiFi

  • RBAC: Role-based access control

✅ How the Internet Works

The internet is like a giant web that connects millions of computers and devices all over the world. It helps us send messages, browse websites, watch videos, and more — all in just a few seconds! But how does it really work behind the scenes? Let’s break it down in a simple way.

🌐 What Is the Internet?

The internet is a global network of computers and servers that share and transfer data using common rules called protocols. It’s like a postal system for digital information.

📦 Packet Switching – Like Sending a Letter in Pieces

When you send a message (like opening a website), the information is broken into small parts called packets. Each packet travels separately across the internet and is reassembled when it reaches its destination — like sending pages of a letter in different envelopes.

🔁 Role of Routers and Switches

Routers are like traffic police. They look at each packet and decide the best path to send it. Switches help connect multiple devices inside the same building or office so they can communicate easily.

⚙️ Internet Protocols (TCP/IP)

Protocols are the rules for how data moves on the internet. The two main ones are:

  • IP (Internet Protocol): It gives every device a unique address (like your home address). It ensures packets reach the correct destination.
  • TCP (Transmission Control Protocol): It checks if all packets arrive safely and in order. If anything is missing, it asks for it again.

🌐 Network of Networks

The internet is not just one big network — it’s made up of thousands of smaller networks like schools, companies, governments, and homes all connected together. This is why we call it a “network of networks.”

📊 Simple Example:

Let’s say you search for “What is cybersecurity?” on Google:

  1. Your phone sends the request in packets to your router.
  2. Your router sends those packets to Google’s server using the internet.
  3. Google replies with search results, again in packets.
  4. Your device reassembles the packets into a readable webpage.

🔑 Key Points to Remember

  • Data is always broken into small packets before transmission.
  • Each device has an IP address (like a digital house number).
  • Routers and switches help guide the data to its correct location.
  • Protocols like TCP/IP make sure everything is delivered safely.

✅ In short, the internet helps your computer talk to other computers anywhere in the world, using smart pathways and common digital languages.

🔥 Firewall & Network Security Devices

A firewall is a security system that acts like a barrier between your private network and the outside world (internet). It filters incoming and outgoing traffic and helps block harmful access.

Firewalls act as gatekeepers that monitor and control incoming and outgoing traffic based on security rules.

🔐 Key Features of a Firewall:

  • Traffic Filtering: Allows or blocks data packets based on rules.
  • Access Control: Prevents unauthorized users from entering the network.
  • Monitoring: Tracks suspicious activity in real-time.
  • Alerts: Sends alerts when threats are detected.

🛡️ Types of Firewalls:

  • Packet-Filtering Firewall: Examines each packet and decides whether to forward it.
  • Stateful Firewall: Tracks active connections and allows only expected traffic.
  • Proxy Firewall: Acts as a gateway between users and the internet.
  • Next-Gen Firewall (NGFW): Advanced firewalls with antivirus, intrusion detection & more.

🔌 Real-World Example:

Imagine your home Wi-Fi router has a firewall. When a hacker tries to enter your system, the firewall blocks that request if it’s suspicious.

💻 Linux Command to View Firewall Status:


sudo ufw status
