Cryptora Q&A

Q1. What is Cryptography?
Cryptography is the practice of securing information by converting it into unreadable text.
Example: Caesar Cipher shifting A→D.
Scenario: Used in online banking to protect transactions.
Q2. Difference between Encryption and Decryption?
Encryption converts plaintext to ciphertext; decryption converts it back.
Example: "HELLO" → "KHOOR" → back to "HELLO".
Scenario: WhatsApp chat messages are end-to-end encrypted.
Q3. What is Symmetric Key Cryptography?
Same key is used for encryption and decryption.
Example: AES (Advanced Encryption Standard).
Scenario: Encrypting a company’s internal files shared among trusted employees.
Q4. What is Asymmetric Key Cryptography?
Uses two keys – a public key for encryption and a private key for decryption.
Example: RSA algorithm.
Scenario: Secure email communication with public/private keys.
Q5. What is a Cipher?
A cipher is an algorithm used for encryption and decryption.
Example: Caesar Cipher, Vigenère Cipher.
Scenario: Substitution ciphers used in puzzle games.
Q6. What is Plaintext and Ciphertext?
Plaintext is the original readable message; ciphertext is the encrypted unreadable text.
Example: "HELLO" (plaintext) → "KHOOR" (ciphertext).
Scenario: Passwords stored in databases as ciphertext.
Q7. What is a Key in Cryptography?
A key is a piece of information that determines the output of a cryptographic algorithm.
Example: AES-256 uses a 256-bit key.
Scenario: Banking apps generate session keys for secure communication.
Q8. What is Hashing?
Hashing is a one-way function that converts input data into a fixed-size output.
Example: SHA-256 hashing.
Scenario: Passwords stored as hashes instead of plain text.
Q9. Difference between Hashing and Encryption?
Encryption is reversible (with key); hashing is irreversible.
Example: AES vs SHA-256.
Scenario: Encryption for chats, hashing for password storage.
Q10. What is Digital Signature?
A digital signature ensures authenticity and integrity of data.
Example: Signed PDF documents.
Scenario: Verifying software updates are from a trusted vendor.
Q11. What is SSL/TLS?
Protocols that secure communication over the internet.
Example: HTTPS websites use SSL/TLS.
Scenario: Online shopping transactions are protected with TLS.
Q12. What is End-to-End Encryption?
Only sender and receiver can read the message, no one in between.
Example: WhatsApp chats.
Scenario: Private messaging apps use E2E encryption.
Q13. What is Steganography?
Hiding information inside other files like images or audio.
Example: Hiding text inside an image.
Scenario: Used in covert communication.
Q14. What is a Man-in-the-Middle Attack?
An attacker intercepts communication between two parties.
Example: Fake Wi-Fi hotspot.
Scenario: Hackers intercept login credentials on public Wi-Fi.
Q15. What is a Brute Force Attack?
Trying all possible keys/passwords until the correct one is found.
Example: Cracking a 4-digit PIN.
Scenario: Hackers brute-forcing weak passwords.
Q16. What is a Dictionary Attack?
Using a list of common passwords to guess the correct one.
Example: Trying "123456" or "password".
Scenario: Attackers crack accounts using leaked password databases.
Q17. What is a Salt in Hashing?
Random data added before hashing to make hashes unique.
Example: Hash(password+salt).
Scenario: Prevents rainbow table attacks on stored passwords.

Q38. What is Fermat’s Little Theorem?
If p is prime and a is not divisible by p, then a^(p−1) ≡ 1 (mod p).
Example: For p=5, a=2 → 2⁴ mod 5 = 1.
Scenario: Used in primality testing for cryptographic keys.
