🔐 Encryption & Cryptography

"Learn how encryption protects your data and keeps your online activity safe. Dive into the heart of cybersecurity — Cryptography!"

Cryptography is the science of securing communication between parties by transforming data into a form that cannot be understood by unauthorized individuals. It plays a central role in protecting information in the digital world — from passwords and online banking to confidential emails.

🔐 What is Cryptography?

Cryptography is the science and technique of securing digital information by converting it into a secret code to prevent unauthorized access. It plays a key role in cybersecurity by protecting data privacy and ensuring safe communication.

🌐 Core Principles of Cryptography:

  • Confidentiality: Only authorized users can view the data.
  • Integrity: Ensures data remains unchanged during transfer.
  • Authentication: Confirms the identity of sender and receiver.
  • Non-repudiation: Prevents anyone from denying that they sent or received the data.

🧠 Types of Cryptography:

There are three major types of cryptography used in data security:

  • Symmetric Key Cryptography: Uses a single secret key for both encryption and decryption. It’s fast and ideal for large amounts of data.
  • Asymmetric Key Cryptography: Uses a pair of keys (public and private). It’s more secure and widely used for secure email, digital signatures, and SSL connections.
  • Hash Functions: Converts data into a fixed-size string (hash) that cannot be reversed. Mostly used in password storage and integrity verification.

Cryptography is the backbone of online security, used in everything from messaging apps and websites to banking and government systems.

🔑 Symmetric Encryption (Private Key)

In symmetric encryption, the same key is used for both encryption and decryption. It is fast and suitable for large data transfers.

Symmetric encryption is a cryptographic method where one secret key is used to both encrypt and decrypt the data. It is widely used due to its speed, simplicity, and effectiveness for large-scale data encryption.

🔑 How It Works:

The sender encrypts the message using a secret key, and the receiver uses the same key to decrypt it. Both parties must have access to this key beforehand, making secure key sharing essential.

✅ Key Characteristics:
  • Uses a single shared key for encryption and decryption
  • Much faster than asymmetric encryption
  • Efficient for encrypting large volumes of data
  • Requires a secure method to share the key
  • Mostly used in closed systems, where key exchange is controlled
🧠 Real-World Examples:
  • Encrypting files with AES-256 in banking and finance
  • WPA2/WPA3 encryption in Wi-Fi routers
  • BitLocker and VeraCrypt for disk encryption
  • SSL/TLS sessions use symmetric encryption after handshake
  • VPNs use symmetric encryption to secure data in transit
🔐 Common Algorithms:
  • AES (Advanced Encryption Standard) – Most widely used today
  • DES (Data Encryption Standard) – Now obsolete due to small key size
  • 3DES (Triple DES) – More secure upgrade of DES
  • Blowfish / Twofish – Flexible and fast encryption methods
💡 Advantages:
  • High speed encryption and decryption
  • Less computational power required
  • Simple implementation
⚠️ Disadvantages:
  • Key distribution is a challenge (secure sharing)
  • If the key is compromised, all data is at risk
  • No built-in sender verification (no authentication)
🖥️ Linux Commands Used in Symmetric Encryption:

# Generate a random 256-bit key using OpenSSL
openssl rand -hex 32

# Encrypt a file using AES-256
openssl enc -aes-256-cbc -salt -in myfile.txt -out encrypted_file.txt

# Decrypt the file
openssl enc -d -aes-256-cbc -in encrypted_file.txt -out decrypted_file.txt
📘 Fun Fact:

During World War II, the German Enigma machine was an early form of symmetric encryption — where both sender and receiver needed to set the machine to the same configuration (shared key).

In modern cybersecurity, symmetric encryption is still the backbone of secure data transmission when speed and efficiency matter most. It is often combined with asymmetric encryption to exchange the secret key securely.

AES is a widely used symmetric encryption algorithm approved by the U.S. government. It encrypts data in blocks of 128 bits using 128, 192, or 256-bit keys.
  • Strong and secure – resistant to known attacks
  • Used in: SSL, VPNs, mobile apps, Wi-Fi (WPA2/WPA3)
  • Recommended for modern encryption

DES was the first major encryption standard. It encrypts data in 64-bit blocks using a 56-bit key.
  • Now considered insecure due to small key size
  • Used in: Legacy systems, ATMs
  • Replaced by: AES and 3DES

Blowfish is a fast and flexible encryption algorithm. It uses a key size between 32 to 448 bits.
  • Free and unpatented
  • Used in: File encryption, bcrypt, secure storage
  • Good alternative to DES

RC4 is a stream cipher that encrypts data one byte at a time. It was once widely used, but is now outdated.
  • Fast and lightweight
  • Used in: Early SSL/TLS, WEP Wi-Fi
  • Not secure – avoid using in modern systems

RC5 is a customizable block cipher with variable block sizes (32, 64, or 128 bits) and key lengths (up to 2040 bits).
  • Highly flexible and efficient
  • Used in: Embedded systems, VPNs
  • Good choice for performance-critical apps

🔐 Asymmetric Encryption (Public Key Cryptography)

Asymmetric encryption uses two different keys: a public key for encryption and a private key for decryption. It’s widely used for secure communication, especially over the internet.

Asymmetric encryption is a powerful cryptographic technique that uses two different keys — a public key to encrypt data and a private key to decrypt it. Unlike symmetric encryption, the sender and receiver do not need to share the same key.
This method enhances security and is widely used in secure web communication, digital signatures, and key exchange protocols.


💡 Key Characteristics of Asymmetric Encryption:

  • Public Key: Shared openly and used to encrypt data.
  • Private Key: Kept secret and used to decrypt data.
  • One-way encryption: Only the private key can decrypt data encrypted with the public key.
  • Use Cases: Secure web browsing (HTTPS), SSH, Digital Signatures, Secure Emails (PGP).
  • Popular Algorithms: RSA, ECC, ElGamal, DSA

📌 Real-World Examples:

  • HTTPS / SSL certificates
  • Email encryption (e.g., PGP)
  • Digital Signatures
  • SSH login authentication
🧪 Example OpenSSL Commands:

# 🔑 Generate RSA Private Key
openssl genrsa -out private_key.pem 2048

# 🌐 Generate Public Key from Private
openssl rsa -in private_key.pem -pubout -out public_key.pem

# 🔐 Encrypt a message using public key
openssl rsautl -encrypt -inkey public_key.pem -pubin -in message.txt -out encrypted.bin

# 🔓 Decrypt message using private key
openssl rsautl -decrypt -inkey private_key.pem -in encrypted.bin -out decrypted.txt

📌 Note: Asymmetric encryption is slower but more secure than symmetric encryption and is often used for key exchange and digital signatures.

RSA is one of the most widely used public-key cryptographic algorithms. It uses a pair of keys — public and private — for secure data transmission.
  • Used for: Secure emails, SSL/TLS, digital signatures
  • Key size: Typically 2048 or 4096 bits
  • Slower than symmetric encryption but more secure for key exchange

ECC is a modern asymmetric algorithm that offers high security with smaller key sizes, making it ideal for mobile and IoT devices.
  • Faster and lighter than RSA
  • Used in: Bitcoin, TLS, secure messaging
  • Example curve: secp256k1 (used in blockchain)

Diffie-Hellman is a key exchange method that allows two parties to establish a shared secret over an insecure channel.
  • Foundation for many modern protocols
  • Used in: VPNs (IPSec), TLS handshake
  • Does not directly encrypt data — only generates a shared key

ElGamal is a public-key encryption scheme based on the Diffie–Hellman principle. It provides both encryption and digital signatures.
  • Probabilistic encryption — produces different ciphertexts each time
  • Used in: PGP, GPG tools
  • Requires more computation and longer ciphertext

🔸 Hashing Algorithms

A hash function is a special algorithm that converts any input (text, file, password) into a unique fixed-length string called a hash value or digest. Hashing is commonly used to protect data integrity and is a core part of cryptography. Unlike encryption, hashing is a one-way process — you can’t reverse it to get the original input.

📌 Key Features of Hash Functions:

  • One-Way: Hash values can't be reversed back to original data.
  • Deterministic: The same input always produces the same hash.
  • Fixed-Length: Hash outputs are always fixed in size, like 256 bits for SHA-256.
  • Collision Resistant: It is hard to find two inputs with the same hash value.
  • Fast Computation: Hashing is quick and ideal for verifying large data files.

🧠 Applications of Hashing in Cybersecurity:

  • ✅ Password Storage: Websites store hashed passwords so even if leaked, they are unreadable.
  • ✅ Digital Signatures: Hashing ensures data was not changed during transmission.
  • ✅ File Verification: Tools like `sha256sum` help verify downloaded files.
  • ✅ Blockchain: Every block uses hashing to connect securely to the previous block.

🔒 Common Hash Algorithms:

  • MD5: Fast but outdated — vulnerable to collisions.
  • SHA-1: Better but no longer recommended for high-security tasks.
  • SHA-256: Part of SHA-2 family, secure and widely used today.
  • BLAKE2: Modern and fast, often used in applications that need speed and security.

💻 Linux Commands for Hashing:


# 🔍 SHA-256 hash of a file
sha256sum filename.txt

# 🔍 MD5 hash of a string
echo -n "hello" | md5sum

# 🔍 SHA-1 hash of a string
echo -n "hello" | sha1sum

# 🔍 SHA-256 hash of a password
echo -n "password123" | sha256sum
📌 Example Use Case:

If you download software like Kali Linux, you’ll see a “SHA256 checksum”. You can run sha256sum filename.iso and compare it to the website’s hash value. If it matches, the file is safe and unaltered.

🎯 Summary: Hash functions protect data from tampering. They’re simple, fast, and powerful — used in almost every area of cybersecurity from passwords to blockchain technology.

🔐 MD5 (Message Digest 5):

  • Output Length: 128-bit hash value (32 characters).
  • Speed: Very fast and lightweight.
  • Security: Weak — vulnerable to collision attacks.
  • Use Cases: File checksums (non-critical), legacy applications.
# Linux Command Example
echo -n "test" | md5sum

🔐 SHA-1 (Secure Hash Algorithm 1):

  • Output Length: 160-bit hash value (40 characters).
  • Security: More secure than MD5, but still broken.
  • Use Cases: Previously used in SSL, GIT, certificates — now deprecated.
# Linux Command Example
echo -n "test" | sha1sum

🔐 SHA-256 (Secure Hash Algorithm 256-bit):

  • Output Length: 256-bit hash value (64 characters).
  • Security: Very strong — currently recommended for secure hashing.
  • Use Cases: Blockchain (Bitcoin), digital signatures, file integrity checks, password hashing.
# Linux Command Example
echo -n "test" | sha256sum

✅ Summary: Use SHA-256 for all security-critical hashing. Avoid MD5 and SHA-1 for anything requiring strong protection.

🔸 Real-World Example: WhatsApp

WhatsApp uses End-to-End Encryption using the Signal Protocol, which is a combination of symmetric and asymmetric encryption. Messages are encrypted on your device and decrypted only on the recipient's device.

# Check GPG installed
gpg --version

# Decrypt the file
